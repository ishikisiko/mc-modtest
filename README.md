# myvillage — Building Library Preview Mod (v0.3.0)

A NeoForge 1.21.1 resource mod that ships a procedurally generated medieval
building library: 30 structures (3 archetypes x 10 seeds) generated by a
layered building pipeline, packed as structure NBT + mcfunctions.

## 1. Build the mod

```bash
./gradlew clean build
```

Requires JDK 21. The build automatically syncs generated resources before
packing the jar.

## 2. Jar output

```text
build/libs/myvillage-0.3.0.jar
```

## 3. Install

Drop the jar into the `mods/` folder of a Minecraft 1.21.1 NeoForge client or
server.

## 4. Test the gallery in game

```mcfunction
/function myvillage:gallery/medieval_village
```

This places all 30 buildings in three archetype rows (28 blocks apart in x,
36 in z). Use a flat world for best results.

## 5. Place a single building

```mcfunction
/place template myvillage:medieval_village/small_house_001 ~ ~ ~
/place template myvillage:medieval_village/medium_house_001 ~ ~ ~
/place template myvillage:medieval_village/blacksmith_001 ~ ~ ~
```

Every structure also has a one-line function:

```mcfunction
/function myvillage:place/small_house_001
```

## 6. Regenerate the building library

```bash
python tools/generate_building_library.py --style medieval_village --count 10
python tools/validate_building_library.py --style medieval_village
```

The generator writes **directly into the mod resources tree** (this project
deliberately skips the `src/generated/resources` indirection so the jar always
matches the last generation run):

```text
src/main/resources/data/myvillage/structure/medieval_village/*.nbt
src/main/resources/data/myvillage/function/gallery/medieval_village.mcfunction
src/main/resources/data/myvillage/function/place/*.mcfunction
reports/building_library_report.json            quality reports per building
reports/building_library_validation.json        NBT-level validation report
```

## 7. Not included in v0.3.0

- worldgen (no structure sets / jigsaw / biome placement)
- Java custom commands
- GUI / config screens
- villagers / entities / spawners
- complex block entity NBT (chest contents, sign text, ...)

## Building generation pipeline (tools/buildgen/)

```text
Style Profile        tools/buildgen/styles/medieval_village.json (+ style.py)
Building Archetype   tools/buildgen/archetypes.py   small_house / medium_house / blacksmith
Scale Tier           tools/buildgen/archetypes.py   small / medium / large_lite
Massing Graph        tools/buildgen/massing.py      main + wing + porch + chimney + shed nodes
Facade Grammar       tools/buildgen/facade.py       bay split, posts, jittered windows, density per face
Build Ops            tools/buildgen/ops.py          hollow_box/wall_frame/gable_roof/doorway/window_kit/...
Pass + Protection    tools/buildgen/passes.py       10 passes, tag/priority grid, PROTECTED cells
Quality Check        tools/buildgen/quality.py      entrance/windows/interior/gables/forbidden blocks + scores
Resource export      tools/buildgen/export.py       structure NBT + gallery/place mcfunctions
```

Key properties:

- materials are resolved through abstract **material slots**
  (`BASE_STONE`, `WALL_MAIN`, `FRAME_WOOD`, `ROOF_DARK`, ...) — building code
  never hardcodes concrete blocks
- archetypes differ functionally: medium_house has a main volume + side wing
  with two roofs and 3 interior zones; the blacksmith has a furnace cluster,
  chimney, open work shed with stone floor, anvil, and barrels
- scale tiers change composition (large_lite = main + wing + rear shed +
  courtyard), never uniform scaling
- failed buildings are rejected by the quality check and regenerated with a
  new seed; reports list errors, warnings, and heuristic scores

# Legacy: Structure JSON -> NBT Pipeline (stages 1-5)

The original minimal pipeline for generating and validating vanilla Minecraft structure NBT files without compiling a NeoForge mod.

The current target verification loop is:

```text
structure JSON -> validate -> convert to .nbt -> export to test world -> /place template
```

## Files

```text
tools/json_to_nbt.py              Convert the custom JSON DSL into gzipped vanilla structure NBT
tools/validate_structure_json.py  Validate JSON DSL operations and blockstate syntax
tools/export_to_world.py          Copy a generated .nbt into a Minecraft test world's generated structures folder
examples/test_house_01.json       First technical validation house, legacy blocks[] DSL
examples/test_house_02.json       Second house, ops[] DSL with fill/set operations and a sloped roof
examples/test_house_03.json       Third house, improved medieval cottage using line/fill/set operations
docs/visual_feedback.md           In-game visual feedback from /place template and screenshot reviews
docs/building_rules.md            Reusable building rules learned from visual feedback
docs/blockstate_notes.md          Notes for blockstates that are easy to get wrong
out/                              Generated .nbt output files
```

## Visual feedback notes

Before asking AI to modify a structure, read these files first:

```text
docs/visual_feedback.md
docs/building_rules.md
docs/blockstate_notes.md
```

Workflow:

```text
1. After a screenshot review or /place template test reveals a visual issue, record it in docs/visual_feedback.md first.
2. If the issue is a reusable building rule, summarize it in docs/building_rules.md.
3. If the issue is related to Minecraft blockstate behavior, summarize it in docs/blockstate_notes.md.
4. Before later AI edits to structures, read all three files before changing JSON or generated NBT.
```

## Supported JSON DSL

The converter remains compatible with the original legacy form:

```json
{
  "blocks": [
    { "pos": [0, 0, 0], "state": "floor" }
  ]
}
```

It also supports the newer operation form:

```json
{
  "ops": [
    { "op": "set", "pos": [4, 1, 1], "state": "door_lower" },
    { "op": "fill", "from": [1, 0, 1], "to": [7, 0, 7], "state": "foundation" },
    { "op": "line", "from": [1, 4, 1], "to": [7, 4, 1], "state": "beam_x" }
  ]
}
```

`state` can reference a palette alias or a direct namespaced blockstate such as:

```text
minecraft:cobblestone
minecraft:spruce_stairs[facing=north,half=bottom,shape=straight,waterlogged=false]
```

## Stage 4: structure template library (Minecraft 1.21.1 only)

Stage 4 adds a reusable template library with village-assembly metadata:

```text
examples/
  buildings/   small_house_01, small_house_02, workshop_01, storage_01
  props/       well_01, lamp_post_01, market_stall_01
  roads/       path_straight_01, path_corner_01
```

Each stage-4 JSON carries a `metadata` block (not written to NBT):

```json
{
  "metadata": {
    "id": "myvillage:small_house_01",
    "category": "building",
    "size": [11, 8, 10],
    "entrances": [{ "pos": [5, 1, 0], "facing": "north" }],
    "weight": 10,
    "tags": ["house", "medieval", "oak", "cobblestone"]
  }
}
```

Rules enforced by the validator:

```text
- metadata.id required, namespaced like myvillage:small_house_01
- category: building | prop | road
- size: [x, y, z], must match the structure size
- building: at least 1 entrance; road: at least 2 connections/entrances
- entrance/connection pos must be inside size; facing: north/south/east/west
- weight: positive number; tags: array of strings
- block ids validated against the 1.21.1 registry (docs/ai-kb/references/blocks_121.json)
```

### Validate a single structure

```bash
python tools/validate_structure_json.py examples/buildings/small_house_01.json --mc-version 1.21.1
```

### Export a single structure

```bash
python tools/json_to_nbt.py examples/buildings/small_house_01.json out/1_21_1/small_house_01.nbt --mc-version 1.21.1
```

### Batch export all templates

```bash
python tools/batch_export.py --mc-version 1.21.1
```

This writes `out/1_21_1/<structure_name>.nbt` for every JSON under
`examples/buildings/`, `examples/props/`, and `examples/roads/`, plus a
summary report at `out/export_report.json` (success/failure counts, and per
structure: id, category, size, block_count, palette_count, output_path,
entrances, connections, block id list).

### Copy to a test world

```bash
python tools/copy_to_world.py --world "<path-to-your-save>"
```

This copies `out/1_21_1/*.nbt` to `<world>/generated/myvillage/structures/`.
Old files are kept by default; add `--clean` to delete previously copied
`.nbt` files first.

### One command: generate all stage-4 structures

`tools/generate_all_buildings.py` builds every structure DSL in Python
(`build_small_house_01()` etc.), writes the JSON files under examples/,
validates them, and can chain export and world copy. It reuses
validate_structure_json.py, batch_export.py, and copy_to_world.py.

Generate all JSON files and validate:

```bash
python tools/generate_all_buildings.py --mc-version 1.21.1 --force
```

Generate, validate, and export NBT:

```bash
python tools/generate_all_buildings.py --mc-version 1.21.1 --force --export
```

Generate, export, and copy into a test world:

```bash
python tools/generate_all_buildings.py --mc-version 1.21.1 --force --export --world "<path-to-your-save>"
```

Clean old NBT files in the world before copying:

```bash
python tools/generate_all_buildings.py --mc-version 1.21.1 --force --export --world "<path-to-your-save>" --clean
```

Flags:

```text
--mc-version  defaults to 1.21.1 (the only supported version)
--export      export NBT to out/1_21_1/ via batch_export.py
--world       copy NBT into <world>/generated/myvillage/structures/ (implies --export)
--clean       remove old .nbt files in the world target directory first
--force       overwrite existing JSON files under examples/ (default: keep)
--seed        reproducible random material variation on roads (default 0)
```

A run report is written to `out/generated_buildings_report.json` (per
structure: metadata, block_count, palette_count, used_blocks, and
validation/export/copy status).

## Current scope

Implemented:

```text
- JSON DSL validation
- set/fill/line operation expansion
- palette alias resolution
- basic blockstate string syntax validation
- duplicate/overwritten coordinate reporting
- vanilla gzipped structure NBT writing
- export helper for the generated/<namespace>/structures folder
- test_house_02 with foundation, logs, walls, glass windows, door, lantern, and sloped spruce roof
- test_house_03 with mixed cobblestone/wood walls, framed windows, interior utility blocks, sealed gables, and a one-block roof overhang
- stage 4 template library (4 buildings, 3 props, 2 roads) with metadata (id/category/size/entrances/connections/weight/tags)
- block id validation against the 1.21.1 registry
- batch export (tools/batch_export.py) with out/export_report.json
- world copy helper (tools/copy_to_world.py)
```

Not implemented yet:

```text
- Per-block blockstate property registry validation (syntax only)
- Structure rotation/mirroring of templates
- Automatic road/village assembly from entrances and connections
- Terrain adaptation for roads and buildings
- Block entity NBT, such as chests, signs, command blocks, spawners
- Entities
- Loot tables
- Schematic or Litematic export
- NeoForge worldgen integration
- Jigsaw/template pool/structure_set generation
```

## 第五阶段预备：批量预览所有建筑

这不是正式的第五阶段村庄拼接，只是第四阶段建筑库的批量验收工具：把所有建筑横向排成一排，一条指令全部摆放，肉眼验收。不含道路布局、地形适配、NeoForge worldgen。

运行脚本：

```bash
python tools/generate_blueprint_gallery_function.py --spacing 20 --axis x
```

参数：`--spacing` 建筑间距（默认 20），`--axis x|z` 排列轴向（默认 x）。
脚本会自动扫描 out/1_21_1/*.nbt，排除文件名含 test/debug/gallery/preview 的临时结构，并输出：

```text
out/datapack_phase5_gallery/                              一个最小 datapack（含 pack.mcmeta）
out/datapack_phase5_gallery/data/myvillage/function/phase5_blueprint_gallery.mcfunction
reports/phase5_blueprint_gallery_manifest.json            清单（function 路径、spacing、axis、每个建筑的 offset 和尺寸）
```

进游戏测试：

```text
1. 先用 tools/copy_to_world.py 把结构 NBT 拷入测试世界（generated/myvillage/structures/）
2. 把 out/datapack_phase5_gallery/ 整个目录拷入 <world>/datapacks/
3. 游戏内执行：
   /reload
   /function myvillage:phase5_blueprint_gallery
```

## NeoForge 1.21.1 最小 Gradle 项目（资源打包型 Mod）

项目现在可以打包成一个 NeoForge 1.21.1 资源型 Mod（只携带 myvillage 结构 NBT 和 mcfunction，不做 worldgen，不注册方块/物品）。

构建（需要 JDK 21）：

```bash
./gradlew build
```

产物：`build/libs/myvillage-0.1.0.jar`，直接放进 mods/ 即可，游戏内可用：

```text
/place template myvillage:small_house_01
/function myvillage:phase5_blueprint_gallery
```

构建前会自动执行 tools/sync_resources_to_mod.py（纯标准库）同步资源：

```text
generated/myvillage/structures/*.nbt                      -> src/main/resources/data/myvillage/structure/
generated/myvillage/data/myvillage/function/*.mcfunction  -> src/main/resources/data/myvillage/function/
```

本 worktree 若没有 generated/myvillage 目录，则自动回退到现有输出位置：
out/1_21_1/*.nbt 和 out/datapack_phase5_gallery/data/myvillage/function/*.mcfunction。

关键文件：build.gradle（ModDevGradle 2.0.141 + NeoForge 21.1.233）、settings.gradle、gradle.properties、
src/main/java/com/example/myvillage/MyVillageMod.java、src/main/resources/META-INF/neoforge.mods.toml。
