## ADDED Requirements

### Requirement: Initiation steles ship as hand-authored mod resources
The two initiation blocks SHALL use the current `myvillage` namespace and SHALL ship hand-authored blockstate, block model, item model, loot table, mineable tool-tag, and bilingual language resources for `myvillage:spirit_testing_stele` and `myvillage:technique_inheritance_stele`. These files SHALL be packaged into the practical mod jar with their registered block and BlockItem classes. This content SHALL NOT be generated by or change the existing structure-NBT exporter, building/compound library generators, place/gallery functions, or structure resource paths.

#### Scenario: The mod jar is inspected
- **WHEN** a release-sensitive build completes
- **THEN** the jar SHALL contain both stele blockstates, block models, item models, loot tables, tool-tag membership, and `en_us`/`zh_cn` entries
- **AND** both Java block/BlockItem registrations SHALL be present

#### Scenario: Structure export is compared before and after the change
- **WHEN** the hand-authored stele resources are added
- **THEN** the structure exporter and generated structure/place/gallery outputs SHALL remain unchanged
- **AND** no stele structure template or worldgen resource SHALL be implied

### Requirement: Usable stele acquisition and command documentation accompany the artifact
The mod SHALL expose both steles in `myvillage:main` and through `/give @s myvillage:spirit_testing_stele` and `/give @s myvillage:technique_inheritance_stele`. README acceptance/usage documentation SHALL list those acquisition commands and the related `awaken`/`juexing` and `initiate`/`rumen` administrator routes before manual review. The resource contract SHALL explicitly state that neither stele has a recipe, natural generation, or sect/worldgen placement in this change.

#### Scenario: A reviewer prepares initiation acceptance
- **WHEN** a current jar is handed off for manual review
- **THEN** README SHALL identify both `/give` commands and all English/pinyin initiation routes
- **AND** it SHALL state that creative inventory or `/give` is the current acquisition path

#### Scenario: Export scope is inspected
- **WHEN** recipes, structure templates, structure sets, biome placement, and sect generation are searched for the stele ids
- **THEN** none SHALL exist in this change
- **AND** the absence SHALL match the documented non-goal rather than a missing promised resource
